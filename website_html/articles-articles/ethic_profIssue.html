Ethical and professional issues in relation to Software Code of Ethics pertaining to software engineers Software engineers

According to Sommerville (2016) software engineers are committed to eight principles which governs their actions and behaviors, these eight principles are public, Client and employer, product, judgement, management, profession, colleagues and self. however there are times when the Principles conflict with the interest of others besides software engineers.

For example public principle, states that the software engineer should have the public’s interest as top priority, but that can change if the senior management gives different instructions, for example the company is working on a certain project and they are under pressure due to time and budget, then the senior manager instructs a software engineer to skip the quality assurance step which determines if the product is fully ready for commercial use.

The software engineer will have to decide whether to follow employment confidentiality by listening to his/her boss or follow the code of ethics and notify the public about the product. These are common challenges that software engineers go through. (Gotterbarn et al, 1997) states that the eight Principles of the Code of Ethics are more of a guidance rather than legislation.

Therefore not all software engineers follow them to the core and in some situations the principles do not complement each other which will require the software engineer to think outside the framework provided by the Code of Ethics and sometimes might end up breaking one of the principles, for instance the principle on public and the principle on client and employer can clash if the software engineer has to develop technology for a client that has the potential to harm others such as nuclear or mass surveillance technology which both can be used for good or bad depending on who intends to use it.
